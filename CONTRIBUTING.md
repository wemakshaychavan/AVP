# Contributing to AVP

Thank you for your interest in contributing to the AI Vendor Protocol!

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your change (`git checkout -b my-feature`)
3. **Make your changes** and commit (`git commit -m 'Add feature'`)
4. **Push** to your branch (`git push origin my-feature`)
5. **Open a Pull Request**

## What to Contribute

- Protocol specification improvements
- SDK implementations in any language
- Documentation fixes and improvements
- Example integrations
- Bug reports and feature requests

## Reporting Issues

Use GitHub Issues. Please include:
- A clear description of the problem
- Steps to reproduce (if applicable)
- Expected vs actual behavior

## Code Style

- Follow existing code conventions in the repository
- Include tests for new functionality
- Update documentation for any protocol changes

## Protocol Changes

Changes to the core protocol specification require:
1. An issue describing the proposed change
2. Discussion and consensus from maintainers
3. A PR with the specification update and rationale
