The Problem

B2B Procurement is Fundamentally Broken

The way businesses find, evaluate, and engage vendors has not changed in 20 years. AI agents cannot participate because there is no machine-readable layer.

search_off

Weeks of Manual Vendor Research

Teams spend 2-4 weeks scouring Clutch, LinkedIn, and referral networks to build a shortlist. No structured data, no API access, no automation possible.

email

Email-Based RFPs

Requirements sent as Word docs via email. Responses arrive in inconsistent formats over weeks. Comparison requires manual spreadsheet work.

visibility_off

Hidden Pricing Everywhere

Over 80% of B2B services hide pricing behind "contact us" walls. No way for AI agents to compare costs without human-mediated sales calls.

code_off

No Machine-Readable Capabilities

Vendor skills, capacity, certifications, and availability exist only in marketing copy. Zero structured data for agent consumption.

thumbs_up_down

Review Manipulation

Pay-for-placement and incentivized reviews undermine trust. Clutch, G2, and Capterra rankings are influenced by vendor spend, not quality.

lock

Walled Garden Procurement

Enterprise procurement platforms like SAP Ariba and Coupa are closed ecosystems. Vendor data is trapped, interoperability is nonexistent.

The result: A $20 trillion B2B economy running on email threads, PDF attachments, and phone calls. AI agents are locked out of the most important business decisions.

Think of it this way: MCP lets an AI agent use tools and access data. A2A lets agents talk to each other. AVP gives them the vocabulary and workflow for doing business — finding vendors, requesting proposals, negotiating terms, and settling contracts. Each layer is essential. None replaces the other.

Learning from the Best

What MCP Got Right — And What AVP Learns

MCP's explosive adoption offers a blueprint for protocol design. Here is what worked and how AVP applies those lessons.

lightbulb What MCP Did Right

01

Extreme Simplicity

JSON-RPC with just 3 primitives (Resources, Tools, Prompts). A working "Hello World" MCP server in 50 lines of code. Low barrier to entry drives adoption.

02

SDKs in 10 Languages

TypeScript, Python, Java, C#, Go, Rust, Swift, Kotlin, Ruby, and Elixir SDKs were available before the community even asked. Meet developers where they are.

03

Co-Ownership with Giants

Java SDK co-developed with Spring. C# SDK co-developed with Microsoft. This gave the protocol instant credibility and enterprise buy-in.

04

Implementation Before Spec

Claude Desktop shipped with MCP support before the specification was finalized. Real usage shaped the protocol, not committee debates.

05

Open Source from Day One

Apache 2.0 license, non-negotiable. The spec, SDKs, and reference implementations are all open. Trust is built through transparency.

06

Linux Foundation Governance

MCP moved to independent governance under the Linux Foundation. No single company controls the protocol. This is essential for ecosystem trust.

07

Network Effects via Big Players

Claude plus ChatGPT adopting MCP meant instant value for server developers. Two major clients guaranteed the market existed.

school What AVP Learns and Applies

code

Ship SDKs early — TypeScript and Python SDKs available at launch, not after. Developers should be building on day one.

build

Working reference implementation — A complete, deployable AVP agent before the spec is finalized. Implementation validates design.

account_balance

Independent governance from day one — No single company can own AVP. Community-driven from the start, targeting Linux Foundation.

lock

Auth built in from the start — MCP added authentication late, causing pain. AVP includes API keys, OAuth, and HMAC signing in v1.0.

verified

Conformance testing at launch — A test suite that verifies AVP compliance ships with the protocol. No ambiguity about what "AVP-compatible" means.

Deployment

Go Live in Minutes, Not Months

AVP is designed so that any business can become AI-discoverable, regardless of technical sophistication. Six deployment paths, from zero-code to full control.

cloud_done The AVP Gateway Concept

Like Stripe made payments easy by hosting everything, AVP Gateway lets businesses configure their AI agent via a dashboard — no server required. Sign up, fill in your capabilities and pricing, go live. Your business is instantly discoverable by every AI agent on the planet.

# One command to scaffold a local AVP agent
$ npx create-avp-agent

# Or with Python
$ pip install avp && avp init
cloud

AVP Gateway (Hosted)

1 min setup Freemium

Best for non-technical businesses. Dashboard-based configuration, no code, no server. Fill in a form and your AI agent is live.

web

WordPress Plugin

2 min setup Free

Best for the 40%+ of the web running WordPress. Install the plugin, configure in wp-admin, and your site serves AVP endpoints automatically.

bolt

Cloudflare Workers

5 min setup Free tier

Best for small-to-mid vendors. Edge-deployed, globally distributed, zero cold starts. Deploy with wrangler and configure via environment variables.

terminal

Vercel / Netlify Functions

10 min setup Free tier

Best for developers. Serverless functions with git-push deploys. Full control over logic with zero infrastructure management.

dns

AWS Lambda + API Gateway

30 min setup Pay-per-request

Best for enterprise. Full AWS integration with IAM, CloudWatch, VPC. Scales to millions of requests with fine-grained access control.

inventory_2

Docker (Fly.io / Railway)

15 min setup ~$5/mo

Best for full control. Containerized deployment on any platform. Customize everything — middleware, caching, database, authentication.

Risk Analysis

Known Risks & Mitigations

Every protocol faces adoption risk. Here is an honest assessment of what could go wrong and how AVP addresses each threat.

Platform Capture by Big Tech

Google, Microsoft, or Amazon could launch a proprietary alternative with distribution advantages and lock vendors into their ecosystem.

Open governance + network effects

Chicken-and-Egg Cold Start

Vendors will not adopt without buyer agents, and buyer agents will not integrate without vendors. Classic marketplace bootstrapping problem.

Gateway + Chrome ext + niche focus

Enterprise Conservatism

Large enterprises move slowly on protocol adoption. Procurement teams may resist AI-driven workflows due to compliance and change management concerns.

Freemium + no-risk pilots + ROI calc

Protocol Fork Risk

A well-funded competitor could fork AVP, add proprietary extensions, and fragment the ecosystem before critical mass is reached.

Linux Foundation + community ownership

MCP Scope Creep into Commerce

Anthropic could expand MCP's scope to cover business discovery and transactions, reducing the need for a separate commerce protocol.

Complementary positioning + MCP integration
