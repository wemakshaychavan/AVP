What is AVP?

A standard way for any website to expose an AI agent that can be discovered, queried, and negotiated with by other AI agents. The protocol ends at proposal acceptance.

The Problem

Finding and evaluating technology vendors is manual, slow, and opaque. Businesses spend weeks researching on Clutch and LinkedIn. RFPs go via email. Responses are inconsistent. Pricing is hidden behind "contact us" walls.

The Solution

AVP defines a protocol so AI agents can handle the entire vendor discovery and negotiation cycle automatically — from finding matching vendors to comparing proposals to accepting the best one.

Think of it as DNS + HTTP + OAuth, but for AI business agents

const { AVPVendor } = require('@avp/vendor-sdk'); const vendor = new AVPVendor({ agentId: 'avp:vendor:your-company:a1b2c3d4', capabilities: { technologies: [ { name: 'java', proficiency: 'expert', years: 10 } ], domains: [ { name: 'travel', experience_years: 8 } ], services: ['custom-development', 'consulting'] } }); // Handle incoming RFPs with your AI vendor.on('rfp', async (rfp, session) => { const proposal = await generateProposal(rfp); return proposal; }); vendor.listen(3000); // ✓ Now discoverable at /.well-known/avp.json
const { AVPBuyer } = require('@avp/buyer-sdk'); const buyer = new AVPBuyer({ agentId: 'avp:buyer:your-company:e5f6g7h8', registryUrl: 'https://registry.avp-protocol.org' }); // Search for vendors matching your needs const vendors = await buyer.search({ technologies: ['java', 'spring-boot'], domains: ['travel'], minTrustScore: 0.7, budget: { max: 200000, currency: 'USD' } }); // Send RFP to top vendors, get ranked proposals const proposals = await buyer.sendRfp(rfp, vendors); const ranked = await buyer.compareProposals(proposals); console.log(ranked[0]); // → { vendor: "Acme Tech", score: 0.92, price: $175K }
from avp_vendor import AVPVendor vendor = AVPVendor( agent_id="avp:vendor:your-company:a1b2c3d4", capabilities={ "technologies": [ {"name": "java", "proficiency": "expert"} ], "domains": [ {"name": "travel", "experience_years": 8} ], "services": ["custom-development"] } ) @vendor.on_rfp async def handle_rfp(rfp, session): proposal = await generate_proposal(rfp) return proposal vendor.run(port=3000) # ✓ Now discoverable at /.well-known/avp.json
# Discover a vendor $ curl https://vendor.com/.well-known/avp.json # Handshake (initiate session) $ curl -X POST https://vendor.com/avp/handshake \ -H "AVP-Version: 1.0" \ -H "AVP-Agent-ID: avp:buyer:you:e5f6g7h8" \ -H "AVP-Signature: ed25519:..." \ -d '{"type": "handshake_request"}' # Send an RFP $ curl -X POST https://vendor.com/avp/rfp \ -H "AVP-Session-ID: ses_abc123" \ -d '{"type": "rfp", "title": "Travel Platform"}' # → Receive structured proposal with pricing

Positioning

Where AVP Fits

AVP is the business communication layer in the AI protocol stack — complementary to MCP and A2A.

AVP — Business Commerce

Discovery, RFP, Negotiation, Proposal Acceptance • AI-to-AI business communication

A2A — Agent Communication

General agent-to-agent task delegation • Google's Agent-to-Agent Protocol

MCP — Tool & Data Integration

Connect AI to tools, databases, APIs • Anthropic's Model Context Protocol

HTTP / HTTPS

Transport layer

AVP doesn't compete with MCP or A2A — it's the specialized business layer that sits above them.

10,000+ vendors. Multiple registries (global, regional, industry). Protocol recognized by industry analysts. Begin formal standardization.
